# Plan 3 — UX & Accessibility Polish

## Context

The UI is solid but has discoverability and accessibility gaps found during exploration.
This plan is a set of small, independent improvements; each can ship alone.

## Items

**1. Consistent transition progress bar (`src/ui.js:160`).**
`show()` sets the bar to `0%` only for `kind === 'author'` and `100%` otherwise, so
hub/category transitions show a full bar with no motion. Make the behavior consistent:
either animate a short indeterminate pulse for hub/category (which load no textures) or
explicitly render them as instant (hide the track when there is nothing to load). Pick the
indeterminate pulse for a smoother feel; CSS-only via a `.transition-bar.indeterminate`
class.

**2. Catalog focus trap + SR announcements (`src/catalog.js`, `index.html:55`).**
The catalog is `role="dialog" aria-modal="true"` but Tab can escape it and view changes
(categories → detail → search results) are silent to screen readers.

- Add a focus trap: on open, cycle Tab/Shift+Tab within `#catalog-panel`; restore focus
  to `#catalog-btn` on close.
- Add an `aria-live="polite"` status region inside the catalog card that announces view
  changes ("Resultados de búsqueda: N", "Sala de <autor>").

**3. Painting focus affordance (`src/ui.js:197-218`, `src/paintings.js`).**
Currently the only feedback that a painting is focused is the info card sliding up. Add a
subtle visual cue on the focused painting's frame (e.g. raise `emissiveIntensity` on the
frame material, which already has `emissive: 0x221a10` at `src/paintings.js:125`). The
focus tracker already resolves the focused interactable group (`findInteractable`,
`src/ui.js:223`) — emit the focused group to a small highlight controller that toggles
emissive on enter/leave.

**4. Favorites discoverability (`src/catalog.js`, `styles.css`).**
The favorite star (`☆`/`★`) is small. Enlarge the affordance in catalog rows and the info
card (`#info-favorite`, `index.html:81`), and ensure the favorites filter button
(`#catalog-favorites-filter`) reads its count (e.g. "Favoritos (3)").

**5. Landscape / mobile info-panel overlap.**
On touch landscape the bottom-center `#info-panel` can collide with the run/info buttons
(`src/controls.js` touch UI). Add a landscape media-query in `styles.css` to shift the
panel or shrink its max-height.

**6. In-play help affordance.**
Onboarding lives only on the welcome screen (`index.html:13-45`). Add a small "?" button
(desktop top-right / reuse mobile button cluster) that re-opens a controls cheatsheet
overlay, so controls are recoverable mid-session.

## Files

- `src/ui.js` (transition bar, focus highlight wiring)
- `src/catalog.js` (focus trap, aria-live, favorites count)
- `src/paintings.js` (frame emissive highlight hook)
- `index.html` (aria-live region, help button markup)
- `styles.css` (indeterminate bar, landscape panel, larger favorite affordance, help overlay)

## Verification

- Keyboard-only: open catalog (M), Tab through — focus stays trapped; Esc closes and
  returns focus to the catalog button.
- Screen reader (VoiceOver): navigate catalog views → confirm announcements fire.
- Focus a painting → confirm subtle frame highlight; look away → highlight clears.
- Resize to mobile landscape → confirm info panel doesn't overlap controls.
- Trigger hub/category/author transitions → confirm progress bar behaves consistently.
- `npm run build` succeeds.

## Sequencing note

Items are independent. Suggested order by value: 1 (quick, visible) → 3 (affordance) →
2 (a11y) → 4 → 5 → 6.
